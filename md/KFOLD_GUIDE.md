# 五折交叉验证使用说明

## 文件说明

- **main.py**: 原始单次训练程序
- **main_kfold.py**: 新增的五折交叉验证程序

## 使用方法

### 运行五折交叉验证

```bash
python main_kfold.py
```

### 运行原始单次训练

```bash
python main.py
```

## 五折交叉验证原理

### 单次划分（原始）
```
全部数据 (n=390)
    ↓
一次划分: 训练70% | 验证10% | 测试20%
    ↓
训练一次 → 得到一个结果
```

### 五折交叉验证（新）
```
全部数据 (n=390)
    ↓
┌─────────────────────────────────────┐
│ Fold 1: 折1做测试，其他做训练         │
│ Fold 2: 折2做测试，其他做训练         │
│ Fold 3: 折3做测试，其他做训练         │
│ Fold 4: 折4做测试，其他做训练         │
│ Fold 5: 折5做测试，其他做训练         │
└─────────────────────────────────────┘
    ↓
5 次训练 → 汇总结果 (均值 ± 标准差)
```

## 关键特性

### 1. 分层抽样 (Stratified)
每个 fold 中保持标签比例一致：
```
全部数据: F0:57, F1:89, F2:59, F3:33, F4:32
Fold 1:  F0:11, F1:17, F2:11, F3:6,  F4:6  (比例相同)
Fold 2:  F0:11, F1:18, F2:12, F3:7,  F4:6
...
```

### 2. 数据划分
每折内部再划分：
```
Fold 的训练数据 (80%)
    ↓
    训练: 90% (用于训练模型)
    验证: 10% (用于早停)
    ↓
Fold 的测试数据 (20%) (用于最终评估)
```

## 训练时间估算

- 单次训练: ~5-10 分钟
- 五折交叉验证: ~25-50 分钟 (5个 loss × 5 folds = 25 次训练)

## 优势

1. **更可靠的评估**: 每个样本都被测试过一次
2. **减少方差**: 5 次结果的平均值更稳定
3. **充分利数据**: 小数据集特别重要

## 建议

- **论文发表**: 必须用五折交叉验证
- **快速实验**: 用单次训练
- **最终结果**: 用五折交叉验证
